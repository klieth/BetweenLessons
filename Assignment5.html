RESTful Interface

HTTP Method URL form Parameters Result Structure Description
GET /user.php None Array of IDs Returns all users in the system.
GET /user.php/uid None {uid: Integer,
firstName: String,
IName: String,
class: Array of Integers
} 		Returns all relevant information about a given user.
POST /user.php/uid delete: [any value] Status of delete Deletes the selected user
GET /song.php uid: id of logged in user Array of IDs Returns all songs related to the logged in user.
GET /song.php/sid status: indicates whether the song is marked "to learn", "mastered", or "current work". {sid: Integer,
title: String,
composer: String,
tempo: Integer,
genre: String,
date: Date,
comments: String,
} 		Returns all relevant information about selected song.
POST /song.php/sid [keyname]: the new value for the specified key

add: [any value] 		None Sets [keyname] to the given value. This will typically only be done to change to comment if not adding a new song, but it is left open to fix spelling errors, etc. in other columns

If adding, all keys should be specified.
POST /song.php/sid delete: [any value] Status of delete Removes a song from a user's list
GET /tasks.php uid: id of logged in user Array of IDs Returns al task related information about songs to the user
POST /tasks.php/tid [keyname]: the new value for the specified key.

add: [any value] 		None Sets [keyname] to the given value. This will be done when the user wisheds to change the status of a song they are learning.

If adding, all keys should be specified
POST /tasks.php/tid delete: [any value] Status of delete Removes a task from a user's list.

User

A User is encoded in JSON as follows:
id: INT, primary key
first: CHAR(25), first name
last: CHAR(25), last name

Song

A Song is encoded in JSON as follows:
sid: INT, primary key
uid: INT, foreign key User(id)
title: CHAR(25), name of song
composer: CHAR(25)
tempo: INT
genre: CHAR(25)
startdate: CHAR(3)
comments: CHAR(25)

Tasks

A Task is encoded in JSON as follows:
tid: IT, primary key
uid: INT, foreign key User(id)
sdate: CHAR(25)
fdate: CHAR(25)
status: CHAR(25)
